IP Subnetting Demystified: From CIDR to Broadcast in 10 Minutes

A hands-on walkthrough of IPv4 subnetting using binary arithmetic. Practice with our free IP Subnet Calculator.

MW

Marcus Webb

Network Engineer · May 29, 2025

Subnetting is one of those skills that every network engineer learns, most forget, and everyone needs when something breaks at 2am. This guide gives you the mental model and the math — once, correctly.

What is CIDR?

CIDR (Classless Inter-Domain Routing) notation expresses an IP address and its associated routing prefix. 192.168.1.0/24 means the first 24 bits are the network portion, and the remaining 8 bits are for hosts.

The Binary Foundation

Every subnet calculation is binary arithmetic. A /24 network has a subnet mask of 11111111.11111111.11111111.00000000 in binary — or 255.255.255.0 in dotted-decimal.

IP Address:     192.168.1.100  →  11000000.10101000.00000001.01100100
Subnet Mask:    255.255.255.0  →  11111111.11111111.11111111.00000000
                                  ─────────────────────────────────────
Network Addr:   192.168.1.0    →  11000000.10101000.00000001.00000000
Broadcast:      192.168.1.255  →  11000000.10101000.00000001.11111111

The Three Key Calculations

  • Network Address = IP AND Subnet Mask (bitwise AND)
  • Broadcast Address = Network OR (NOT Mask) — all host bits set to 1
  • Usable Hosts = 2^(32−prefix) − 2 (minus network and broadcast)

Variable Length Subnet Masking (VLSM)

VLSM allows you to split a network into subnets of different sizes — allocating address space efficiently. A /24 network can be split into multiple /26 networks (62 hosts each), /28 networks (14 hosts), and /30 point-to-point links — all without wasting address space.
